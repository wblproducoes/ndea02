Crie um sistema completo em Node.js (Express) compatível com servidores cPanel, usando MySQL/MariaDB como banco de dados. O sistema deve gerenciar registros contendo os campos:

- Série (texto)
- Aluno (texto)
- Mesa (texto)
- Veio? (Sim/Não)
- Chamado? (Sim/Não)
- Produtos (texto)

## FUNCIONALIDADES OBRIGATÓRIAS

### 1) Interface e Tabelas
- Exibir registros em “tables”, cada um com um identificador único (nome do table).
- O nome do table deve permanecer o mesmo mesmo após recarregar a página.
- O nome do table **só pode ser alterado** ao digitar a senha: **Iesg510@** ( e quando aparecer o campo de digitar a senha de um autofocus no campo).

### 2) Alternância de Status (Sim/Não)
- O campo **Veio?** e o campo **Chamado?** devem alternar entre “Sim” e “Não” ao clicar.
- Cada mudança deve:
  - Ser gravada em banco de dados (tabela de logs).
  - Acionar sincronização em tempo real para todos os clientes conectados.
  - Exibir no registro qual table fez a alteração (ex.: “Alterado por: Sala A”).

### 3) Carregamento via Excel (IMPORTAÇÃO)
- Deve existir recurso para fazer upload de um arquivo **Excel (.xlsx)** com colunas:
  `Série`, `Aluno`, `Mesa`, `Veio`, `Chamado`, `Produtos`.
- Só permite upload quando for digitada a senha **Iesg510@**.
- Após upload:
  - Inserir/atualizar registros no banco.
  - Garantir que o table tenha um identificador persistente.
  - Atualizar todos os clientes via tempo real.

### 4) Reset Geral
- Um botão “RESETAR” redefine todo o estado (ex.: coloca Veio e Chamado em “Não”).
- O reset só funciona ao digitar a senha **Iesg510@**.
- Registrar o reset na tabela de logs.

### 5) Contadores por Série
- No topo da interface mostrar:
  - Quantos alunos “Vieram” por série.
  - Quantos “Chamado” por série.
- Valores atualizam em tempo real.

### 6) Sincronização em Tempo Real
- Usar Socket.IO (ou WebSocket) para:
  - Notificar mudanças de status.
  - Notificar upload de Excel.
  - Notificar reset.
  - Notificar renomeação de table.

### 7) Logo e Tela Cheia
- Exibir o logo:
  `https://saogoncalosp.com.br/wp-content/uploads/2025/10/logo_novo_h_350x100px.svg` 
- Criar um botão “Tela Cheia”.

## BANCO DE DADOS (MySQL/MariaDB)

Criar ao menos três tabelas:

### Tabela: tables
- table_id (PK)
- table_name
- created_at

### Tabela: records
- id (PK)
- table_id (FK)
- serie
- aluno
- mesa
- veio (boolean)
- chamado (boolean)
- produtos (text)
- updated_at

### Tabela: changes (log)
- id (PK)
- record_id
- table_id
- field_name
- old_value
- new_value
- changed_at
- changed_by (IP ou identificador do table)

Cada alteração de Veio/Chamado, upload, reset ou rename deve ir para esta tabela.

## ENDPOINTS A IMPLEMENTAR

- POST /admin/upload-excel  
  - file (xlsx), password  
  - Atualiza DB e notifica via Socket.IO.

- POST /admin/reset  
  - password  
  - Reseta o estado e notifica.

- POST /admin/rename-table  
  - table_id, new_name, password  
  - Atualiza nome e notifica.

- GET /api/records?table_id=
  - Lista registros.

- POST /api/record/:id/toggle
  - body: { field }  // field = "veio" ou "chamado"
  - Alterna o valor, salva log e notifica.

- GET /api/counters
  - Retorna contadores por série.

- GET /api/tables
  - Lista tables e nomes.
